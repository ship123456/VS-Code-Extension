## Introduction

This guide provides solutions to common issues that users may encounter while installing or using the Technical Documentation Generator extension.

Follow the recommended solutions before contacting support.

---

## Common Issues

| Issue | Recommended Solution |
|--------|----------------------|
| Extension not found | Verify the extension name and search again in the Extensions Marketplace. |
| Installation failed | Check your internet connection and retry the installation. |
| Extension not activated | Restart Visual Studio Code and reopen the project. |
| Command not available | Open the Command Palette and verify the extension is installed and enabled. |
| Documentation not generated | Ensure a project folder is open and all required fields are completed. |

---

## Installation Problems

### Extension cannot be installed

Possible causes:

- Internet connection unavailable
- Marketplace temporarily unavailable
- Unsupported VS Code version

Recommended solution:

- Verify your internet connection.
- Update Visual Studio Code.
- Retry the installation.
- Install using a `.vsix` package if Marketplace access is restricted.

---

## Activation Problems

### Extension commands do not appear

Possible causes:

- Extension failed to activate
- Workspace not opened
- Installation incomplete

Recommended solution:

1. Restart Visual Studio Code.
2. Open a project folder.
3. Verify the extension is enabled.
4. Reinstall the extension if necessary.

---

## Documentation Generation Issues

### Documentation is not generated

Possible causes:

- Required fields are empty.
- No documentation template selected.
- Workspace is not open.

Validation behavior:

- Required fields are highlighted when left empty.
- Inline validation messages identify missing or invalid input.
- Documentation generation is blocked until all required fields are completed.

Recommended solution:

- Complete all required fields.
- Select a documentation template.
- Open a project folder before generating documentation.
- Try generating the document again after correcting validation errors.

---

## Viewing Extension Logs

If the extension fails to generate documentation or behaves unexpectedly, diagnostic information can be viewed in Visual Studio Code.

### Developer Tools Console

1. Open **Help → Toggle Developer Tools**.
2. Select the **Console** tab.
3. Review any error messages or stack traces generated by the extension.

### Output Panel

1. Open the **Output** panel.

**Windows / Linux**

```text
Ctrl + Shift + U
```

**macOS**

```text
Cmd + Shift + U
```

2. From the Output dropdown, select **Technical Documentation Generator**.
3. Review the extension logs for warnings or error messages that may help identify the issue.

## File Management Issues

### Generated file cannot be found

Possible causes:

- File saved to another folder.
- File was renamed.
- Save operation was cancelled.

Recommended solution:

- Search the current workspace.
- Regenerate the documentation.
- Save the file again.

---

## Performance Issues

### Extension responds slowly

Possible causes:

- Large workspace
- Low system memory
- Multiple extensions running simultaneously

Recommended solution:

- Close unused projects.
- Restart Visual Studio Code.
- Disable unnecessary extensions.
- Restart the computer if required.

## Frequently Asked Questions

### Why is the Generate command unavailable?

Verify that the extension is installed, enabled, and a project workspace is currently open.

### Can I edit generated documentation?

Yes. Generated Markdown files can be modified at any time using Visual Studio Code.

### Does the extension require internet access?

Only for downloading or updating the extension from the Marketplace. Documentation generation works locally.

### Where are generated files stored?

Generated documentation is saved within the current project workspace.

## Getting Support

If the issue persists after following this guide:

- Restart Visual Studio Code.
- Reinstall the extension.
- Review the User Guide and Installation Guide.
- Open the Developer Tools Console or Output Panel to collect diagnostic information.
- Contact the project maintainer with details of the issue, Visual Studio Code version, operating system, and any relevant error messages.
